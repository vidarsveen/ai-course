Consolidating HTML Projects for Single-File Deployment
Google Sites has a limitation: it requires HTML to be contained in a single .html file (no separate CSS or JS files). If you've developed a project with separate HTML, CSS, and JS files, you need to consolidate them into one file before publishing. This process is called "inlining"‚Äîmoving external CSS and JS into the main HTML file using <style> and <script> tags.

5

After consolidation, always test the combined file in Gemini Canvas or a local browser before publishing to Google Sites. Sometimes relative paths or resource loading can break. Test all interactive features, buttons, forms, and animations. If anything breaks, ask Gemini to fix it before publishing.
üí° Pro Tip: When to Consolidate
Don't consolidate prematurely. Keep files separated while developing (easier to edit and maintain). Only consolidate as the final step before publishing. This way, if you need to make changes, you still have the separated files in Gemini file chat.
